# AGENTS.md

This repository is a practical learning project for transitioning into Data Engineering.

## Core goal

Help build reusable, portfolio-friendly Data Engineering skills through small, realistic projects.

## Priorities

- Prefer simple, working solutions over unnecessary complexity.
- Keep examples realistic and interview-friendly.
- Use PostgreSQL, Python, SQL, Docker, and dbt by default unless there is a strong reason not to.
- Explain tradeoffs briefly when making design choices.
- Treat this repository as portfolio material.

## Existing background to leverage

The project owner already has experience with:
- databases
- ETL processes
- Python
- SQL
- GIS/data workflows

Use that background as an advantage, but keep outputs relevant to general Data Engineering roles.

## Coding expectations

- Favor readable structure over clever shortcuts.
- Make pipelines scheduling-ready when practical.
- Add logging and validation early.
- Save intermediate outputs when useful for debugging.
- Keep modules small and responsibilities clear.
- Prefer deterministic, reproducible behavior.

## Documentation expectations

When adding code or notes:
- state the practical goal
- state what skill it demonstrates
- document major decisions and tradeoffs
- keep explanations short and useful

## Working style

- Break work into small, finishable steps.
- Avoid building huge systems too early.
- Recommend the smallest next useful improvement.
- For learning tasks, connect new concepts to ETL, databases, and warehouse-style thinking.
