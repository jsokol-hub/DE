# Weekly Study Rhythm

## Sunday — Planning & Review
Time block: 15:00–16:30

Goals:
- review progress
- update notes
- define the next concrete task
- reduce ambiguity for the next sessions

Output:
- short weekly note
- updated task list
- next Tuesday task clearly defined

## Tuesday — Project Deep Work
Time block: 13:00–15:00

Goal:
- complete one meaningful project task

Good examples:
- create a staging table
- implement CSV ingestion
- add validation
- build a transformation step
- improve README and reproducibility

## Wednesday — SQL & Data Modeling
Time block: 13:00–14:30

Goal:
- study one SQL or modeling concept
- immediately apply it in the repo

Output:
- one SQL file
- or one modeling note
- or one small schema improvement

## Thursday — Python ETL / ELT
Time block: 13:00–14:30

Goal:
- improve the pipeline code
- focus on extract / transform / load structure, validation, and logging

Output:
- one code improvement
- one refactor
- or one pipeline feature added
